# Sasaya AI - 產品需求文件 (PRD)

## 1. Summary

### 產品概述
Sasaya AI 是一個 Apple Mac App桌面應用程式，旨在將用戶分散在各處的 LLM AI 聊天記錄轉換為結構化的個人知識庫。透過一鍵匯入 AI 聊天記錄、自動分類和智慧搜尋，幫助用戶將過去的思考轉化為可搜尋、可組織的筆記系統。

### 核心價值主張
1. **一鍵轉換**：將 ChatGPT 聊天記錄轉換為結構化筆記，保留完整上下文
2. **自動組織**：使用 AI 自動分析內容並生成分類，讓想法不再散落
3. **智慧搜尋**：用戶可以在個人知識庫中快速找到任何想法和見解
4. **隱私優先**：所有資料儲存在用戶的電腦本地端，不上傳到任何雲端服務

### 目標用戶
- 經常使用 ChatGPT 或其他 AI 工具進行深度對話的人，可能是企業專業人士，也可能是個人工作者。
- 需要管理大量 AI 對話記錄的知識工作者。
- 希望建立個人知識庫的學習者和研究者。

### MVP 範圍
- 不包含帳號登入/註冊功能
- 專注於核心功能：匯入  ChatGPT 聊天記錄 zip 檔案、轉換成筆記、自動分類
- 目標：30 分鐘內可開發出可運作的 prototype

## 2. 功能需求

### 2.1 匯入功能
- **功能描述**：用戶可以匯入 ChatGPT 匯出的 .zip 檔案
- **支援格式**：標準 .zip 壓縮檔（包含 ChatGPT 匯出的 JSON 格式聊天記錄）
- **處理流程**：
  1. 用戶選擇 .zip 檔案 
  2. 系統解壓縮並解析 JSON 格式
  3. 將每個聊天轉換為筆記
  4. 顯示匯入進度
  5. 匯入完成後顯示統計資訊（如：匯入了 X 個對話、建立了 Y 個筆記）

### 2.2 筆記轉換功能
- **功能描述**：自動將 AI 聊天記錄轉換為結構化筆記
- **轉換規則**：
  - 保留對話的完整上下文（用戶問題 + AI 回答）
  - 提取對話標題作為筆記標題（如果 ChatGPT 匯出包含標題）
  - 保留時間戳記
  - 將對話內容格式化為易讀的筆記格式

### 2.3 自動分類功能
- **功能描述**：使用 AI 分析筆記內容並自動生成分類
- **分類邏輯**：
  - 分析筆記的主題、關鍵字和內容語義
  - 自動將相似主題的筆記歸類到同一分類
  - 生成有意義的分類名稱（如：產品管理、AI 與科技、個人成長等）
- **分類顯示**：在左側欄以資料夾圖示和名稱顯示

### 2.4 手動編輯功能
- **分類管理**：
  - 創建新分類
  - 編輯分類名稱
  - 刪除分類
  - 將筆記移動到不同分類
- **筆記管理**：
  - 編輯筆記標題
  - 編輯筆記內容（支援基本格式化：粗體、斜體、底線等）
  - 刪除筆記
  - 在分類間移動筆記

### 2.5 搜尋功能（MVP 版本）
- **功能描述**：在筆記中進行關鍵字搜尋
- **搜尋範圍選擇**：
  - 所有筆記
  - 特定分類（資料夾）
  - 特定筆記
- **搜尋行為**：
  - 即時搜尋（輸入時即顯示結果）
  - 高亮顯示匹配的關鍵字
  - 顯示搜尋結果列表（標題 + 預覽片段）

### 2.6 本地儲存
- **功能描述**：所有資料儲存在用戶的 Mac 本地
- **儲存位置**：應用程式專屬的本地資料庫或檔案系統
- **資料內容**：
  - 筆記內容
  - 分類結構
  - 筆記與分類的關聯
  - 用戶編輯記錄

### 2.7 空狀態處理
- **首次使用**：顯示歡迎筆記，介紹產品功能和使用方式
- **無筆記狀態**：顯示匯入提示和操作指引
- **無分類狀態**：顯示「未分類」分類，並提示可創建新分類

## 3. 頁面結構

### 3.1 整體佈局
應用程式採用三欄式佈局（可選兩欄，右側 AI 對話區在 MVP 可簡化或省略）：

```
┌─────────────────────────────────────────────────────────┐
│ [視窗控制] [Integrations] [搜尋/提問/整理] [Note][Ask][Split][Organize] │
├──────────┬──────────────────────────┬───────────────────┤
│          │                          │                   │
│  左側欄   │      主內容區              │   右側 AI 對話區   │
│          │                          │   (MVP 可選)      │
│  - 分類  │  - 筆記編輯工具列          │  - Sasaya AI     │
│  - 筆記  │  - 筆記標題               │  - 對話介面       │
│  列表    │  - 筆記內容               │  - 搜尋範圍選擇    │
│          │  - 格式化工具             │                   │
│          │                          │                   │
└──────────┴──────────────────────────┴───────────────────┘
```

### 3.2 左側欄（Sidebar）
- **頂部操作區**：
  - `+` 按鈕：新增筆記
  - `山匯入` 按鈕：匯入 ChatGPT .zip 檔案
  - 搜尋框：「搜尋/提問/整理」
- **分類列表**：
  - 「未分類」（Uncategorized）
  - 自動生成的分類（如：個人成長、財經與投資、AI 與科技等）
  - 每個分類顯示：
    - 圖示（資料夾圖示）
    - 分類名稱
    - 展開/收合箭頭
- **筆記列表**：
  - 顯示當前分類下的所有筆記
  - 每個筆記顯示：
    - 筆記標題（截斷過長標題）
    - 時間戳記（如：01:35）
  - 當前選中的筆記高亮顯示
- **底部**：設定按鈕（齒輪圖示）

### 3.3 主內容區（Main Content Area）
- **工具列**：
  - 格式化按鈕：B（粗體）、I（斜體）、U（底線）、S（刪除線）、`</>`（程式碼）、Aa（字體選項）
  - 「AI 對話」按鈕（MVP 可選）
- **筆記標題**：可編輯的文字欄位
- **筆記內容**：
  - 可編輯的富文本區域
  - 支援基本格式化
  - 顯示對話轉換後的內容
- **空狀態**：
  - 歡迎筆記（首次使用）
  - 匯入提示（無筆記時）

### 3.4 右側 AI 對話區（Optional for MVP）
- **頂部**：
  - Sasaya AI 圖示和標題
  - 「您的智能知識助手」副標題
  - 關閉按鈕（X）
- **對話區域**：
  - 顯示 AI 的歡迎訊息和功能介紹
  - 顯示對話歷史（MVP 可簡化）
- **輸入區**：
  - 「搜尋範圍」選擇器：所有筆記 / 此資料夾 / 此筆記
  - 輸入框：「詢問任何問題...」
  - 發送按鈕（紙飛機圖示）
  - 提示文字：「按 Enter 發送，Shift + Enter 換行」

### 3.5 頂部導航欄
- **左側**：標準 macOS 視窗控制（紅、黃、綠）
- **中間**：
  - Integrations 圖示（拼圖圖示）
  - 搜尋/提問/整理輸入框（支援 ⌘P 快捷鍵）
- **右側**：
  - Note 按鈕
  - Ask 按鈕
  - Split 按鈕
  - Organize 按鈕

### US-2: 自動將聊天轉換為筆記

**我想要** 系統自動將聊天記錄轉換為結構化筆記

## 5. 技術細節

### 5.1 資料儲存
- **儲存方案**：
  - 選項 1：使用 SQLite 資料庫（推薦，易於查詢和管理）
  - 選項 2：使用 JSON 檔案（簡單，但查詢效率較低）
- **資料結構**：
  ```json
  {
    "categories": [
      {
        "id": "cat_001",
        "name": "產品管理",
        "icon": "chart",
        "createdAt": "2025-01-01T00:00:00Z"
      }
    ],
    "notes": [
      {
        "id": "note_001",
        "title": "筆記標題",
        "content": "筆記內容...",
        "categoryId": "cat_001",
        "createdAt": "2025-01-01T00:00:00Z",
        "updatedAt": "2025-01-01T00:00:00Z",
        "sourceChatId": "chat_001"
      }
    ]
  }
  ```
- **儲存位置**：`~/Library/Application Support/SasayaAI/` 或應用程式沙盒目錄

### 5.2 檔案格式處理
- **ChatGPT 匯出格式**：
  - 標準 .zip 壓縮檔
  - 解壓縮後包含 JSON 格式的聊天記錄
  - JSON 結構範例（需根據實際 ChatGPT 匯出格式調整）：
    ```json
    {
      "conversations": [
        {
          "id": "chat_001",
          "title": "對話標題",
          "createdAt": "2025-01-01T00:00:00Z",
          "messages": [
            {
              "role": "user",
              "content": "用戶問題"
            },
            {
              "role": "assistant",
              "content": "AI 回答"
            }
          ]
        }
      ]
    }
    ```
- **處理流程**：
  1. 驗證 .zip 檔案格式
  2. 解壓縮檔案
  3. 解析 JSON 結構
  4. 轉換為筆記格式
  5. 儲存到本地資料庫

### 5.3 錯誤處理
- **匯入錯誤**：
  - 檔案格式不正確：顯示「無法識別的檔案格式，請確認這是 ChatGPT 匯出的 .zip 檔案」
  - 檔案損壞：顯示「檔案損壞，無法讀取」
  - 匯入過程中斷：顯示「匯入失敗，請重試」
- **儲存錯誤**：
  - 磁碟空間不足：顯示「儲存空間不足，請清理磁碟空間」
  - 權限問題：顯示「無法寫入資料，請檢查應用程式權限」
- **搜尋錯誤**：
  - 無搜尋結果：顯示「未找到匹配的內容」
  - 搜尋失敗：顯示「搜尋時發生錯誤，請重試」

### 5.4 空狀態處理
- **首次使用**：
  - 顯示預設的歡迎筆記
  - 歡迎筆記內容包含產品介紹和使用指引
  - 左側欄顯示「未分類」分類
- **無筆記狀態**：
  - 主內容區顯示匯入提示
  - 提示文字：「點擊左側的「匯入」按鈕，匯入您的 ChatGPT 聊天記錄」
  - 顯示匯入按鈕或操作指引
- **無分類狀態**：
  - 所有筆記顯示在「未分類」分類下
  - 提示用戶可以創建新分類

### 5.5 自動分類 AI 整合（MVP 後續實作）
- **MVP 階段**：可先使用簡單的關鍵字匹配或規則分類
- **後續版本**：整合 AI API（如 OpenAI API）進行語義分析
- **分類邏輯**：
  - 分析筆記內容的主題
  - 提取關鍵字和概念
  - 與現有分類進行相似度匹配
  - 生成新分類或歸類到現有分類

### 5.6 搜尋功能實作（MVP 版本）
- **搜尋方式**：關鍵字全文搜尋
- **搜尋範圍**：
  - 筆記標題
  - 筆記內容
- **搜尋演算法**：
  - 簡單的字串匹配（MVP）
  - 可選：不區分大小寫
  - 可選：支援部分匹配
- **效能考量**：
  - 對於大量筆記，考慮建立索引
  - 使用背景執行，避免阻塞 UI

### 5.7 UI/UX 細節
- **深色主題**：應用程式採用深色模式設計
- **響應式設計**：支援視窗大小調整
- **快捷鍵**：
  - ⌘P：開啟搜尋/提問
  - ⌘N：新增筆記
  - ⌘S：儲存筆記
  - ⌘F：搜尋（在當前筆記中）
- **載入狀態**：
  - 匯入時顯示進度條
  - 處理大量資料時顯示載入動畫
- **確認對話框**：
  - 刪除分類時確認
  - 刪除筆記時確認
  - 匯入大量資料時確認

## 6. 開發 Phases

### Phase 1: 核心基礎架構（優先級：最高）
**目標**：建立基本的應用程式框架和資料儲存

**任務**：
1. 建立 Mac 應用程式專案結構
2. 實作本地資料儲存（SQLite 或 JSON）
3. 建立基本的 UI 框架（三欄式佈局）
4. 實作左側欄（分類和筆記列表）
5. 實作主內容區（筆記顯示和編輯）
6. 實作基本的資料模型（Category、Note）

**預估時間**：10-15 分鐘

### Phase 2: 匯入和轉換功能（優先級：高）
**目標**：實作 ChatGPT 匯入和筆記轉換

**任務**：
1. 實作 .zip 檔案選擇和讀取
2. 實作 JSON 解析（ChatGPT 匯出格式）

## 7. 技術堆疊建議

### 開發框架
- **SwiftUI**（推薦）：Mac 原生開發，UI 構建快速
- **Electron**（備選）：跨平台，但檔案較大

### 資料儲存
- **SQLite**（推薦）：輕量、高效、易於查詢
- **Core Data**（備選）：macOS 原生，但較複雜
- **JSON 檔案**（最簡單）：適合 MVP，但查詢效率低

### 檔案處理
- **Foundation Framework**：處理 .zip 解壓縮和 JSON 解析

### AI 整合（後續版本）
- **OpenAI API**：用於自動分類和 AI 對話
- **本地 AI 模型**（備選）：如 Core ML，但需要預訓練模型
