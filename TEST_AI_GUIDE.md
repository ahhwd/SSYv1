## 🧪 測試場景

### 測試 1：基本問答 ✅

**步驟**：
1. 在右側 AI 面板的輸入框輸入：`什麼是系統一？`
2. 按 **Enter** 發送（或點擊 📤 按鈕）
3. 觀察「思考中」動畫（三個跳動的點點）
4. 等待 1.5 秒

**預期結果**：
- ✅ 看到你的問題出現（綠色泡泡，靠右）
- ✅ 看到「思考中」動畫
- ✅ 看到 AI 回答出現（深灰泡泡，靠左）
- ✅ 回答下方顯示「📎 相關筆記」區塊（如有相關筆記）
- ✅ 自動滾動到最新訊息

---

### 測試 2：探索不同問題類型 🎯

試試這些問題，觀察不同的回答模式：

#### A. 定義型問題（「是什麼」）
```
什麼是系統一和系統二？
什麼是 BEAMS 品牌？
```

**預期回答格式**：
- 摘要
- 詳細說明
- 引用筆記來源

---

#### B. 方法型問題（「如何」）
```
如何提升個人成長？
如何學習新技能？
怎麼提升寫作能力？
```

**預期回答格式**：
- 步驟列表
- 關鍵要點
- 基於過去經驗的建議

---

#### C. 原因型問題（「為什麼」）
```
為什麼要持續學習？
為何創業很困難？
```

**預期回答格式**：
- 原因分析
- 相關脈絡
- 深層思考

---

#### D. 一般問題
```
AI 技術的發展
日本的設計風格
個人成長的關鍵
```

**預期回答格式**：
- 內容摘要
- 關聯筆記列表
- 可點擊的相關筆記

---

### 測試 3：相關筆記功能 📎

**步驟**：
1. 提出任一問題（如：`什麼是系統一？`）
2. 等待 AI 回答
3. 查看回答下方的「📎 相關筆記」區塊
4. 點擊任一相關筆記

**預期結果**：
- ✅ 顯示最多 3 個相關筆記
- ✅ 每個筆記顯示標題和分類
- ✅ Hover 時有視覺效果（藍色邊框 + 位移）
- ✅ 點擊後左側編輯器跳轉到該筆記
- ✅ 該筆記的分類自動展開

### 測試 5：輸入控制 ⌨️

### 測試場景 C：關鍵字匹配精度
提出以下問題，測試搜尋準確度：

**高相關度問題**（應該找到筆記）：
- `系統一` → 應該找到包含「系統一」的筆記
- `BEAMS` → 應該找到品牌相關筆記
- `個人成長` → 應該找到成長類筆記

**低相關度問題**（可能找不到）：
- `火星探險` → 很可能找不到相關筆記
- `量子力學` → 除非有相關筆記

## ⚠️ Demo 版本特性提醒

### 這是 Mock 版本
- **不是真正的 AI**：回答是預先設計的模板
- **無 API 呼叫**：不需要 OpenAI API key
- **關鍵字匹配**：搜尋基於簡單的關鍵字匹配
- **固定模板**：回答格式相對固定

### Mock 的好處
- ✅ 無需 API key 和費用
- ✅ 立即可用，快速展示
- ✅ 測試 UI 和互動流程
- ✅ 驗證使用者體驗

### 與真實 AI 的差異
| 功能 | Demo 版本 | 真實 AI（Phase 5）|
|------|-----------|-------------------|
| 回答內容 | 模板化 | 真實理解 |
| 語義理解 | ❌ | ✅ |

| 成本 | 免費 | 需付費 |
| 精準度 | 中等 | 高 |

## 🐛 已知限制

### 搜尋限制
- 只能匹配關鍵字，無法理解同義詞
- 例如：問「notebook」找不到「筆記本」

### 回答限制
- 回答格式相對固定
- 無法處理複雜的多輪對話
- 不會記住之前的對話上下文

### UI 限制
- 對話歷史不會持久化（重新載入會消失）
- 沒有匯出對話功能
- 沒有清除對話功能
